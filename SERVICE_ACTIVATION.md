# 火山引擎服务开通说明

## 问题分析

根据控制台显示：
- 服务状态：**运行中** ✅
- 服务开通：显示"开通"按钮（红色高亮）⚠️
- 403 错误：认证失败

## 可能的原因

### 1. 服务需要完全激活 ⭐ 最可能

**现象：**
- 点击"开通"按钮后，服务可能还在激活过程中
- 需要等待几分钟让服务完全就绪
- 激活期间 API 可能返回 403

**解决方案：**
1. 点击"开通"按钮后，**等待 5-10 分钟**
2. 刷新控制台页面，确认服务状态
3. 确认"服务开通"列显示为"已开通"（而不是"开通"按钮）
4. 重新测试连接

### 2. 认证信息需要刷新

**可能情况：**
- 开通服务后，Access Token 可能需要重新生成
- 或者需要等待认证信息同步

**解决方案：**
1. 在控制台查看"服务接口认证信息"
2. 确认 Access Token 是否变化
3. 如果变化，更新 `.env.local` 文件
4. 重新启动代理服务器

### 3. 服务权限问题

**检查：**
- 确认账户有使用该服务的权限
- 确认试用包是否已激活
- 确认用量额度是否充足（显示 0.00/20.00 小时）

## 排查步骤

### 步骤 1：确认服务状态

1. 登录火山引擎控制台
2. 进入"豆包流式语音识别模型2.0"服务页面
3. 检查"服务开通"列：
   - ✅ **已开通**：服务可以使用
   - ⚠️ **开通**按钮：服务未完全激活

### 步骤 2：等待服务激活

如果显示"开通"按钮：
1. 点击"开通"按钮
2. **等待 5-10 分钟**
3. 刷新页面
4. 确认状态变为"已开通"

### 步骤 3：验证认证信息

1. 在控制台查看"服务接口认证信息"
2. 确认：
   - APP ID: `5825385546`
   - Access Token: 完整值（点击显示按钮）
3. 更新 `.env.local` 文件（如果需要）

### 步骤 4：重新测试

1. 更新认证信息后，重启代理服务器：
```bash
npm run proxy
```

2. 运行 Python 测试：
```bash
cd sauc_python
python3 test_connection.py
```

3. 如果 Python 测试成功，前端也应该可以工作

## 常见问题

### Q: 点击开通后多久可以使用？

A: 通常需要 **5-10 分钟** 让服务完全激活。请耐心等待并刷新页面确认状态。

### Q: 开通后还是 403 怎么办？

A: 
1. 确认服务状态为"已开通"
2. 检查 Access Token 是否正确
3. 尝试重新生成 Access Token（如果有选项）
4. 联系火山引擎技术支持
